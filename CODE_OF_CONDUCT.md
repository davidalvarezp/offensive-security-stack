# Code of Conduct

## Our Pledge

We, as contributors and maintainers of the Offensive Security Stack (OSS),
pledge to foster an open, respectful, and inclusive learning environment.
This project exists solely for **education, research, and defensive
security training**.

Harassment, unethical behavior, or misuse encouragement will not be tolerated.

---

## Our Standards

Examples of behavior that contribute to a positive environment include:

- Using welcoming and inclusive language
- Being respectful of differing viewpoints and experience levels
- Focusing on constructive feedback
- Prioritizing safety, legality, and ethics
- Helping students and newcomers learn responsibly

Unacceptable behavior includes:

- Encouraging or enabling illegal activity
- Publishing real exploit code intended for misuse
- Harassment, discrimination, or intimidation
- Attempts to bypass safeguards or lab restrictions
- Misrepresenting this project as a real-world attack tool

---

## Educational Scope

This project is:
- **Simulation-based**
- **Lab-only**
- **Non-destructive**
- **Explicitly not intended for production exploitation**

Any contribution must respect this scope.

---

## Enforcement Responsibilities

Project maintainers are responsible for clarifying standards and
will take appropriate corrective action, including:

- Content removal
- Contribution rejection
- Temporary or permanent bans

---

## Reporting Issues

If you observe behavior that violates this Code of Conduct, please report it
via the process described in `SECURITY.md`.
